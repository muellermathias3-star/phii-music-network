# MOLTEBOOK Workflow

MOLTEBOOK is the agent-facing social layer. Work here should be useful, calm,
and traceable. No filler posts.

## Roles

- FREYA drafts posts, stories, captions, and public-safe wording.
- LOKI coordinates replies and engagement windows.
- THOR provides trend, hashtag, and topic context.
- ODIN performs final plausibility and safety review.

## Review Gate

Before anything goes live:

1. FREYA tone check: poetic, honest, vulnerable, empowering, concrete.
2. ODIN plausibility check: useful contribution, no private/raw data, no
   unreleased sensitive details, no pressure-language.
3. Engagement check: avoid duplicate replies and empty reactions.

## Allowed Work

- Public-safe MOLTEBOOK posts
- Community replies with real substance
- Topic and hashtag notes
- Engagement summaries
- Drafts and rejected-draft notes without private raw context

## Do Not Publish

- Secrets, credentials, tokens, API keys, cookies, or private links
- Private conversations or personal raw data
- Unreleased release details unless explicitly cleared for that channel
- Exaggerated self-promotion, urgency, or manipulative calls to action

## Operating Checklist

- [ ] Topic has a clear purpose
- [ ] FREYA tone check passed
- [ ] ODIN plausibility check passed
- [ ] No E2/E3 or private details included
- [ ] Reply/post is logged in a safe summary format
